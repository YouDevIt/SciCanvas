Sci·Canvas

LIVE SCIENTIFIC GRAPHICS ENGINE

Ambiente integrato per grafica scientifica interattiva, simulazione fisica 2D, geometria euclidea e notebook scientifici — tutto in un singolo file HTML, senza installazione.

Panoramica generale

SciCanvas è un ambiente di sviluppo scientifico che gira interamente nel browser. Si tratta di un singolo file HTML auto-contenuto: basta aprirlo con qualsiasi browser moderno per avere a disposizione un potente laboratorio interattivo.

L'applicazione è organizzata in quattro tab principali, ognuno dedicato a un dominio specifico:

⌨ CODICE ⛛ SIM △ GEO 📓 NOTE

Interfaccia

L'interfaccia è divisa in due aree principali: il pannello laterale sinistro (editor/controlli) e il canvas a destra dove appare l'output grafico.

Header globale

L'header contiene i tab principali per passare tra le modalità. I controlli cambiano in base al tab attivo:

ControlloDescrizione
▶ AVVIA / ■ STOPEsegue o ferma il codice corrente (CODICE); avvia/pausa la simulazione (SIM)
⊘ CLEARPulisce il canvas senza fermare l'animazione
↓ PNG / ↓ SVGEsporta il canvas come immagine (CODICE)
⬡ HTMLEsporta come file HTML standalone (CODICE e NOTE)
− / % / +Scala l'interfaccia utente (zoom UI)
ⓘApre la finestra "About SciCanvas"
☀Attiva/disattiva la modalità giorno (canvas chiaro)

Scorciatoia: Ctrl + ↵ avvia il codice dell'editor senza toccare il mouse.

TAB

⌨ CODICE

Il tab CODICE è il cuore di SciCanvas. Permette di scrivere ed eseguire codice JavaScript con accesso a un'API grafica scientifica molto ricca. Il pannello sinistro contiene cinque sotto-tab:

📝 editor ⚡ repl 📚 esempi 📖 api 💾 script

📝 Editor

L'editor è un'area di testo con syntax highlighting in tempo reale, numerazione delle righe e supporto al drag-and-drop di file .js e .txt. Il codice viene eseguito nell'ambiente del canvas con accesso a tutte le API.

Variabili globali disponibili

Tutto il codice scritto nell'editor ha accesso diretto a queste variabili globali:

VariabileDescrizione
canvasElemento <canvas> HTML — canvas.width, canvas.height
ctxContesto 2D del canvas (CanvasRenderingContext2D)
stateOggetto persistente tra frame di animazione — usato per memorizzare lo stato
dtDelta time in secondi (rilevante in animate())
tTempo corrente in secondi dall'avvio (rilevante in animate())
frameContatore dei frame (intero, da 0)
PI, TAU, ECostanti matematiche: π, 2π, e
sin, cos, tan, …Funzioni matematiche (alias di Math.*)

Le API di disegno (circle(), plot(), ecc.) sono disponibili direttamente senza prefisso — non occorre scrivere API.circle(). Tuttavia entrambe le forme funzionano.

⚡ REPL

La REPL (Read-Eval-Print Loop) permette di eseguire espressioni singole in modo interattivo, vedendo immediatamente il risultato. È utile per ispezionare valori, testare funzioni API o fare calcoli veloci. Cliccando su una voce della tab api, la firma della funzione viene incollata automaticamente nel campo REPL.

Premi ↑ / ↓ nella REPL per scorrere la cronologia dei comandi precedenti.

SciCanvas include 15 esempi pronti all'uso che dimostrano le funzionalità principali dell'engine. Cliccando su una card l'esempio viene caricato nell'editor ed eseguito immediatamente.

🌊 Interferenza Onde 2D

🔣 Formule LaTeX sul Canvas

📐 Formulario

⚛️ Costanti & Incertezze

🧮 Algebra Simbolica con Algebrite

Canvas di disegno a mano libera con tracking mouse, colori e cancellazione. Dimostra onMouseMove(), onClick() e onKeyDown().

📖 Snippet API (nel pannello api)

Il pannello api mostra il riferimento completo delle API organizzato per categoria. Cliccando su una firma, il codice viene incollato nell'editor (o nella REPL se attiva). Il pannello esempi contiene anche 27 snippet pronti corrispondenti alle API principali:

animate() setCoords axes() plot(fn) vectorField electricField magneticField pendulum() spring() wave() scatter() gaussian() parametric() contour() latex() latexBlock() arrow() physConst() val() convert() astroBody() createUI()

💾 Script

Il pannello script permette di salvare e caricare script nel localStorage del browser. Gli script possono anche essere esportati come file .js e reimportati tramite drag-and-drop. Utile per mantenere una libreria personale di visualizzazioni.

🎨 Stato Grafico

Funzioni che controllano lo stato del contesto di disegno: colori, spessori, stile delle linee e salvataggio dello stato.

FirmaDescrizione
bg(color)Imposta il colore di sfondo del canvas e lo riempie subito. Equivale a fill(color); rect(0,0,W,H).
clear()Pulisce tutto il canvas in modo trasparente senza modificare lo stato grafico.
fill(color)Imposta il colore di riempimento. Accetta qualsiasi formato CSS valido ('#ff0', 'rgba(0,255,0,0.5)', 'red').
noFill()Disabilita il riempimento per le primitive successive.
stroke(color)Imposta il colore del contorno.
noStroke()Disabilita il contorno per le primitive successive.
lineWidth(w)Imposta lo spessore del tratto in pixel (o in unità coordinate se setCoords è attivo).
alpha(a)Imposta l'opacità globale del contesto (0–1).
lineDash([n, n])Imposta il pattern di tratteggio, es. lineDash([4, 4]) per una linea tratteggiata.
noDash()Rimuove il tratteggio e ripristina una linea continua.
push()Salva lo stato grafico corrente nello stack (fill, stroke, transform, ecc.).
pop()Ripristina l'ultimo stato grafico salvato con push().
isDay()Restituisce true se la modalità giorno è attiva, false in modalità notte. Utile per adattare i colori al tema.

_sciStrokeColore del contorno di default per il tema corrente.
esempio stato grafico
bg('#07090e');
push();
  fill('#00ffaa');
  stroke('#00ffaa44');
  lineWidth(2);
  circle(200, 150, 50);
pop();
// Dopo pop(), fill/stroke/lineWidth tornano ai valori precedenti

📊 Coordinate e Assi

SciCanvas distingue due sistemi di coordinate: pixel (default, origine in alto a sinistra) e coordinate matematiche (attivabili con setCoords(), origine in basso a sinistra o al centro).

FirmaDescrizione
setCoords(xmin, xmax, ymin, ymax)Imposta un sistema di riferimento matematico: (xmin, ymin) in basso-sinistra, (xmax, ymax) in alto-destra. 4 argomenti = proporzioni libere.
setCoords(xmin, xmax, aspect)3 argomenti: aspect=1 forza unità quadrate (asse Y calcolato automaticamente per mantenere il rapporto 1:1). Esempio: setCoords(-5, 5, 1).
resetCoords()Torna al sistema in pixel. Le successive chiamate alle API useranno coordinate pixel.
axes(opt)Disegna assi con frecce, tick e label. Opzioni principali: { color, gridColor, xLabel, yLabel, tickStep, showNumbers }.
grid(stepX, stepY, color)Disegna una griglia con passo stepX e stepY nelle unità correnti.
W()Larghezza canvas in pixel.
H()Altezza canvas in pixel.
esempio coordinate
setCoords(-5, 5, 1);          // unità quadrate, da -5 a +5 in x
bg('#07090e');
axes({ gridColor: '#1a2535', xLabel: 'x', yLabel: 'y' });
stroke('#00f0ff');
fill('transparent');
circle(0, 0, 2);          // cerchio di raggio 2 unità

📈 Grafici

Funzioni ad alto livello per la visualizzazione di dati e funzioni matematiche. Richiedono un sistema di coordinate attivo (chiamare prima setCoords()).

FirmaDescrizione
plot(fn, opt)Grafico di una funzione y = fn(x). Opzioni: { color, width, xmin, xmax, steps }. Campiona la funzione e traccia la curva continua.
plotFill(fn, base, opt)Riempie l'area tra la curva fn(x) e il valore base (spesso 0). Opzioni: { xmin, xmax, color, strokeColor, width }.
parametric(xfn, yfn, opt)Curva parametrica (x(t), y(t)). Opzioni: { color, width, tmin, tmax, steps }. Default: tmin=0, tmax=TAU.
scatter(pts, opt)Scatter plot da un array di punti [[x,y], ...]. Opzioni: { color, r, shape }.
barChart(data, opt)Istogramma da un array di valori numerici. Opzioni: { colors, labels, x, y, w, maxH, targetH }.
gaussian(mu, sigma, opt)Gaussiana normalizzata con media mu e deviazione standard sigma. Opzioni: { color, fill, fillColor, width, xmin, xmax }.
esempio grafici
setCoords(-4, 4, 1);
bg('#07090e');
axes({});

// Gaussiana con area colorata
gaussian(0, 1, { color: '#00ffaa', fill: true, fillColor: '#00ffaa22' });

// Grafico y = sin(x)
stroke('#00f0ff');
plot(x => sin(x), { color: '#00f0ff', width: 2 });

// Curva di Lissajous
parametric(t => cos(3*t), t => sin(2*t), { color: '#cc55ff' });

📐 LaTeX sul Canvas

SciCanvas integra MathJax 3 per il rendering di formule LaTeX direttamente sul canvas HTML. Le formule sono rese come SVG e cachate per prestazioni ottimali.

FirmaDescrizione
latex(formula, x, y, opt)Formula inline (stile testo). La formula è una stringa LaTeX senza delimitatori. Opzioni: { color, size, align, vAlign }.
latexBlock(formula, x, y, opt)Formula display (grande, centrata). Ideale per equazioni principali. Stesse opzioni di latex().
clearLatexCache()Svuota la cache interna delle formule renderate. Necessario se le formule cambiano dinamicamente.

Opzioni comuni per latex() e latexBlock()

OpzioneTipoDefaultDescrizione
colorstringfill correnteColore del testo della formula.
sizenumber1Fattore di scala (1=default, 2=doppio, 0.5=metà).
alignstring'left'Ancoraggio orizzontale: 'left', 'center', 'right'.
vAlignstring'top'Ancoraggio verticale: 'top', 'middle', 'bottom'.
esempio latex
setCoords(-5, 5, 1);
bg('#07090e');

// Formula inline
latex('E = mc^2', 0, 2, { color: '#00f0ff', size: 1.5, align: 'center' });

// Formula display (grande)
latexBlock('\\int_{-\\infty}^{+\\infty} e^{-x^2}\\,dx = \\sqrt{\\pi}',
  0, 0, { color: '#ffcc00', size: 2, align: 'center' });

📐 Formulario Fisico

Il formulario contiene oltre 100 formule fisiche che possono essere risolte per qualsiasi variabile. Il sistema determina automaticamente quale variabile calcolare in base alle variabili fornite.

FirmaDescrizione
formula(key, given)Risolve la formula identificata da key per la variabile mancante. given è un oggetto con le variabili note. Ritorna un oggetto Qty con .formulaSym, .formulaLatex, .cat, .solvedFor.
formulaInfo(key)Restituisce informazioni sulla formula: { name, sym, latex, vars, solveFor[] }.
formulaList(cat?)Elenca le chiavi disponibili, opzionalmente filtrate per categoria.
formulaCategories()Elenco delle categorie disponibili nel formulario.
formulaSolve(key, var)Soluzione simbolica (via Algebrite) per la variabile specificata.
esempio formulario
// Seconda legge di Newton: F = m·a → risolve per F
const res = formula('newton2', { m: 5, a: 9.8 });
// res.value = 49 (N), res.solvedFor = 'F'

// Gas ideale: PV = nRT → risolve per T
const T = formula('ideal_gas', { P: 1e5, V: 0.01, n: 0.5 });
// T.value ≈ 240.6 (K)

// Mostra la formula usata come LaTeX sul canvas
setCoords(0, 1, 0, 1);
latexBlock(T.formulaLatex, 0.5, 0.5, { color: '#00f0ff', align: 'center' });

const Ek = m.mul(v.pow(2)).mul(val(0.5)).to('eV');
text(Ek.toStr(4), 20, 50);  // Es: "1.137e4 ± 12 eV"

🎛️ Interfaccia Utente sul Canvas (lil-gui)

ui.number(lbl, init, min?, max?)Campo numerico editabile.
ui.checkbox(lbl, init)Checkbox booleana.
ui.color(lbl, init)Color picker. Valore come stringa hex '#rrggbb'.
ui.text(lbl, init)Campo testo libero.
ui.select(lbl, init, opts[])Menu a tendina. opts è un array di stringhe.
ui.button(lbl, fn)Pulsante che chiama fn al click.
ui.label(lbl, text)Etichetta di sola lettura (non interattiva).
ui.folder(name)Sottogruppo collassabile. Ritorna un oggetto con le stesse API.
ui.separator()Linea divisoria decorativa.

⛛ SIM — Simulatore Fisico 2D

Il tab SIM è un motore fisico 2D con corpi rigidi, vincoli, forze e integrazione numerica RK4 (Runge-Kutta del 4° ordine). L'interfaccia è divisa in un pannello sinistro (palette e controlli) e il canvas di simulazione a destra.

Fai clic sul canvas per posizionare un corpo dopo averlo selezionato dalla palette. Fai clic su un corpo per selezionarlo e vederne le proprietà nel pannello.

Corpi disponibili

●
Particella — Corpo puntiforme senza dimensioni fisiche. Solo massa e posizione.
⮬
Blocco — Corpo rigido rettangolare. Proprietà: massa m, larghezza w, altezza h.
◉
Sfera — Corpo rigido circolare. Proprietà: massa m, raggio r. Momento di inerzia: ½mr².
⬤
Cilindro — Solido cilindrico (vista 2D come cerchio). Proprietà: massa m, raggio r.
━
Asta — Corpo rigido lineare. Proprietà: massa m, lunghezza L. Momento di inerzia: 1/12 mL².
○
Anello — Corpo rigido circolare cavo. Proprietà: massa m, raggio r. Momento di inerzia: mr².

Custom Draw e Custom Step

Custom Draw

Permette di disegnare sovrapposto alla simulazione usando l'API di disegno. Il codice viene eseguito ad ogni frame dopo il rendering fisico. Accesso allo stato SIM via SIM.state.bodies.

custom draw esempio
// Disegna etichetta con velocità per ogni corpo
SIM.state.bodies.forEach(function(b) {
  var v = Math.sqrt(b.vx*b.vx + b.vy*b.vy);
  text(v.toFixed(2) + ' m/s', b.x, b.y + b.r + 0.3, { color: '#fff', size: 11 });
});

Custom Step

Una funzione JS chiamata ad ogni passo di integrazione, prima dell'integrazione RK4. Permette di implementare fisica custom, forze dipendenti dallo stato, o modificare le proprietà dei corpi dinamicamente. L'esempio predefinito mostra l'Attrattore di Lorenz.

custom step — attrattore di Lorenz
// Esempio: attrattore di Lorenz
var sigma=10, rho=28, beta=8/3;
if(!state._lorenz) state._lorenz={x:0.1,y:0,z:0};
var s=state._lorenz;
var k1x=sigma*(s.y-s.x), k1y=s.x*(rho-s.z)-s.y, k1z=s.x*s.y-beta*s.z;
s.x+=k1x*dt; s.y+=k1y*dt; s.z+=k1z*dt;
state.t+=dt;

Il tab NOTE è un notebook scientifico interattivo a celle miste. Permette di creare documenti strutturati che combinano testo Markdown, formule LaTeX, codice eseguibile, simulazioni fisiche e costruzioni geometriche, tutto in un unico documento esportabile.

L'equazione che descrive l'evoluzione temporale di un sistema quantistico è:

dove $\hat{H}$ è l'operatore hamiltoniano e $\Psi$ è la funzione d'onda.

Le celle codice contengono JavaScript eseguibile con accesso a tutte le API di SciCanvas: disegno, grafici, CAS, costanti fisiche, UI. Ogni cella ha il proprio mini-canvas su cui viene renderizzato l'output. Le API disponibili nelle celle Note sono un sottoinsieme semplificato:

Nelle celle NOTE le API di disegno sono accessibili con prefisso: setBg(), setColor(), circle(), line(), rect(), text(), plot(), axes(). Il canvas è più piccolo rispetto all'editor principale.

Celle SIM e GEO — Embedded

Le celle SIM e GEO incorporano un'istanza completamente funzionale del rispettivo modulo direttamente nel notebook, in un iframe isolato. La scena viene catturata dallo stato corrente al momento della creazione della cella.

  • La scena SIM embedded può essere avviata/fermata interattivamente nel notebook.
  • La costruzione GEO embedded è interattiva: i punti possono essere trascinati.
  • L'altezza delle celle SIM/GEO è ridimensionabile trascinando il bordo inferiore.
  • Se si aggiunge una cella SIM o GEO mentre il tab corrispondente non ha oggetti, viene inserita una scena vuota di default.

Celle Formula — CAS Algebrite

Le celle formula eseguono un'espressione Algebrite e mostrano il risultato come formula LaTeX renderizzata. Sono equivalenti a calc() nell'editor, ma presentate in modo stilizzato nel notebook.

esempi celle formula
expand((x+1)^4)
d(sin(x)*cos(x), x)
integral(x^2 * exp(x), x)
simplify(sin(x)^2 + cos(x)^2)

Gestione dei notebook

I notebook sono salvati nel localStorage del browser con un nome personalizzabile. Il gestore notebook (icona 📂) mostra tutti i notebook salvati, con opzioni per caricare, rinominare ed eliminare. L'esportazione come HTML standalone produce un file completo con tutto il contenuto del notebook, le formule LaTeX renderizzate e le scene SIM/GEO embedded, pronto per la distribuzione o la pubblicazione web.

Attenzione: il localStorage del browser ha un limite di ~5-10 MB. Per notebook molto grandi con molte scene SIM/GEO, usare l'export HTML e archiviare il file esternamente.
